MySQL的扩展：
    复制：每个节点都有相同的数据集；
        M/S主从结构，向外扩展的解决方案
        从节点请求主节点的二进制日志于本地并执行replay
        单向进行

复制的功用：
    数据分布；
    负载均衡读操作；
    数据备份；
    高可用和故障切换；
    MySQL升级测试；

主从复制：
    从节点：
        I/O Thread：从MASTER请求二进制日志时间，并保存于中继日志中；
        SQL Thread：从中继日志中读取日志事件，在本地完成重放；
        
    主节点：
        dump Thread：为每个Slave的I/O Thread启动一个dump线程，用于向其发送binary日志；
    
    特点：
        1、异步复制；
        2、主从数据不一致比较常见；

    复制架构：
        M/S，M/M，环状复制
        一主多从
        级联
        一从多主：属于不同数据库
    
    二进制日志事件记录格式：
        STATEMENT
        ROW
        MIXED
